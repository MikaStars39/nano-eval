## BUG #1: [HIGH] init_ray ignores address parameter

**File**: `nanoeval/ray/utils.py:13-24`

**Description**: The `address` parameter is accepted by the function signature but never passed to `ray.init()`. The `--ray-address` CLI option has no effect — the code always starts a local Ray instance.

```python
# Before
def init_ray(address: str = "auto", **kwargs: Any) -> None:
    ray.init(
        ignore_reinit_error=True,
        include_dashboard=False,
        logging_level="ERROR",
        **kwargs,              # address is NOT in kwargs
    )
```

**Impact**: Users who pass `--ray-address` to connect to an existing Ray cluster silently get a local instance instead. Multi-node distributed runs fail.

**Fix**: Added `address=address` to `ray.init()`.

## BUG #2: [HIGH] Offline workers silently drop failed items

**File**: `nanoeval/backend/offline.py:85-107` (also `_multi_turn_worker` at lines 175-216)

**Description**: When `_worker` encounters an exception during generation, it logs the error and calls `task_done()` but does NOT put the item into `output_queue`. The item is silently lost.

```python
# Before
try:
    ...
    await self.output_queue.put(item)     # only on success
except Exception as e:
    logger.error(f"Worker Error: {e}")    # item dropped here
finally:
    self.input_queue.task_done()
```

**Impact**: Failed inference items are silently lost. The progress bar shows incomplete progress, and the output file has fewer records than expected with no error trace.

**Fix**: Moved `output_queue.put(item)` into the `finally` block. Added `_status` and `_error` fields to failed items, matching the online engine's behavior.

## BUG #3: [HIGH] GPQA substring matching false positives

**File**: `nanoeval/reward/gpqa/gpqa_verify_reward.py:68`

**Description**: For GPQA/MMLU, labels are typically single letters (A, B, C, D). The substring check `"a" in pred_ans.lower()` produces false positives for any prediction containing that letter anywhere.

```python
# Before
if label.lower() in pred_ans.lower():
    return {"pred": pred_ans, "pass": True}

# Examples of false positives:
# Label "A", prediction "Data" -> "a" in "data" is True
# Label "B", prediction "Bob"  -> "b" in "bob" is True
```

**Impact**: Inflated accuracy scores on GPQA/MMLU benchmarks.

**Fix**: Replaced with normalized equality comparison — strip parentheses, whitespace, and uppercase both sides before comparing.

## BUG #4: [MEDIUM] `<think>` blocks not stripped before scoring

**File**: `nanoeval/reward/score.py:12, 52-88`

**Description**: The `_split_response_and_thinking` function handles `</thinking>` and `<|end_of_thought|>` patterns but does NOT handle the `<think>...</think>` format (used by DeepSeek and others). A regex `_THINK_BLOCK_PATTERN` was defined at line 12 but never used.

```python
# Defined but unused:
_THINK_BLOCK_PATTERN = re.compile(r"<think>(.*?)</think>", flags=re.IGNORECASE | re.DOTALL)
```

**Impact**: When a model outputs `<think>reasoning</think>final answer`, the thinking content is not extracted and ends up as part of the response text sent to the judge.

**Fix**: Added `<think>...</think>` handling using the existing `_THINK_BLOCK_PATTERN` regex before the split-pattern loop.

## BUG #5: [MEDIUM] extract_answer return type mismatch

**Files**:
- `nanoeval/reward/math/math_verify_reward.py:13`
- `nanoeval/reward/gpqa/gpqa_verify_reward.py:3`

**Description**: Both `extract_answer` functions are annotated `-> str` but return `None` when no `\boxed{}` content is found.

**Impact**: Incorrect type annotation misleads static analysis tools and IDE users. No runtime crash since callers check via truthiness.

**Fix**: Changed annotation to `-> Optional[str]`.

## BUG #6: [MEDIUM] Online retry delay grows without bound

**File**: `nanoeval/backend/online.py:148-171`

**Description**: `AsyncClient.post_request` uses exponential backoff with `retry_delay *= 2` but no maximum cap. With `max_retries=500`, even 20 consecutive failures produce a delay of 2^20 seconds (~12 days).

```python
# Before
retry_delay = 1.0
for attempt in range(self.config.max_retries + 1):
    ...
    await asyncio.sleep(retry_delay)
    retry_delay *= 2                   # no cap
```

**Impact**: A burst of transient server errors (e.g., 10 consecutive 429s) causes increasingly long waits, effectively hanging the pipeline.

**Fix**: Added `max_retry_delay = 60.0` and capped with `min(retry_delay * 2, max_retry_delay)`.

## BUG #7: [LOW] Destructive regex in merge_and_extract.py

**File**: `recipes/llm_judge/merge_and_extract.py:19`

**Description**: The third pattern `r'.*?\n\n'` with `re.DOTALL` matches everything from the beginning of the text to the first double-newline, stripping legitimate content rather than just thinking blocks.

```python
# Before
patterns = [
    r'<think>.*?</think>',
    r'<\|begin_of_thought\|>.*?<\|end_of_thought\|>',
    r'.*?\n\n',                           # matches from start to first double-newline
]
```

**Impact**: Legitimate score content before the first paragraph break could be removed, causing extraction failures.

**Fix**: Removed the `r'.*?\n\n'` pattern.
